Base de Datos no relacional

Vamos a utilizar como referencia mongoDB.

En esta nueva era, lo que más importa es el TIEMPO. Lo que queremos siempre es que nuestras bases de datos sean lo más rápidas posibles. Esto da origen a la Base de Datos no relacional.

Origen de NoSQL
- Google (big table) y Amazon(dinamoDB)

Bases de Datos NoSQL
Permiten manejar datos no estructurados o semiestructurados, es decir que no necesariamente tienen que estar relacionados y hacer cosas complejas. No tiene esa estructura, esa formalidad que sí tiene una base de datos SQL.
Esto permite recolectar la información de manera mucho más rápida. Mejor performance.

SQL es muy estructurado. Lo que tiene es que cuando le pedimos info. tenemos que pedirle muchas tablas, referencias, relaciones, etc. Y eso hace que la lectura sea más lenta que la escritura. En SQL la escritura es rápida, pero no así la lectura.


SQL 	    ---> ACID (Atomicity, Consistency, Isolation, Durable)
NoSQL ---> BASE (Basically Available, Soft State, Eventual Consistency)


NoSQL es más flexible y sencillo de usar, por no estar arraigado a una estructura tan marcada como SQL.
